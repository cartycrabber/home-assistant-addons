# Home Assistant App: PostgreSQL 18

PostgreSQL 18 is a powerful, open-source relational database system. This app
runs the official `postgres:18` Docker image managed by the HA Supervisor
lifecycle, with native HA backup integration and full SSL/TLS support.

## First Start

On the very first start, PostgreSQL initializes the data directory (`initdb`).
This takes a few seconds. After initialization the `postgres_user`,
`postgres_password`, and `postgres_db` options **cannot be changed** by simply
editing the app configuration â€” they are baked into the database cluster at
init time.

To change credentials after initialization you must use SQL (`ALTER ROLE`) or
wipe the data directory (destroying all data) and reinitialize.

## Configuration Options

| Option              | Default            | Description                                                 |
| ------------------- | ------------------ | ----------------------------------------------------------- |
| `postgres_user`     | `postgres`         | Superuser username                                          |
| `postgres_password` | _(required)_       | Superuser password â€” masked in the HA UI                    |
| `postgres_db`       | _(empty)_          | Initial database name; defaults to `postgres_user` if empty |
| `initdb_args`       | `--data-checksums` | Arguments passed to `initdb` on first start only            |
| `ssl`               | `false`            | Enable SSL/TLS encryption for all connections               |
| `ssl_certfile`      | `fullchain.pem`    | SSL certificate filename in the HA `/ssl/` directory        |
| `ssl_keyfile`       | `privkey.pem`      | SSL private key filename in the HA `/ssl/` directory        |
| `extra_args`        | `[]`               | Additional `-c name=value` PostgreSQL config parameters  |

## Connecting to PostgreSQL

The default host port is **5432**. You can change or disable the port mapping
in the app's _Network_ settings.

Connection string example:

```
postgresql://postgres:<password>@homeassistant.local:5432/postgres
```

From an app run by the same Supervisor, connect via the Supervisor-managed
internal hostname `local_postgresql18` on port 5432 (HA resolves app slugs on
the internal Docker network). From outside the host, use your HA machine's
network hostname or IP (e.g., `homeassistant.local`).

## SSL / TLS Encryption

To enable encrypted connections:

1. Set `ssl: true` in the app configuration.
2. Ensure your HA SSL directory contains the certificate and key files
   referenced by `ssl_certfile` and `ssl_keyfile` (typically generated by the
   _Let's Encrypt_ integration: `fullchain.pem` and `privkey.pem`).
3. Restart the app.

PostgreSQL will enforce a minimum of **TLSv1.2**. All connections â€” including
local â€” will be offered SSL. Clients can still connect without SSL unless you
add `hostssl all all 0.0.0.0/0 scram-sha-256` to `pg_hba.conf` via the
advanced configuration method below, replacing the default `host` rule.

## Backup and Restore

### Automatic (HA Snapshots)

This app uses HAOS's **cold backup**: HA stops the app, snapshots the entire data
volume (which includes the full PG18 cluster at `18/docker/` and the app
options), then restarts. The snapshot is consistent and requires no extra
configuration.

> **Note**: cold backup means the database is briefly unavailable during HA snapshot creation.

### Manual SQL Dumps (External Backups)

For portable SQL backups or pre-migration dumps, run `pg_dump` or `pg_dumpall` from a remote machine with `psql` installed:

```bash
pg_dumpall -h homeassistant.local -U postgres > backup.sql
```

### Restore from SQL Dump

```bash
psql -h homeassistant.local -U postgres -f backup.sql
```

## Data Integrity

`--data-checksums` is enabled by default via the `initdb_args` option. This
enables page-level checksums that detect silent data corruption (bit flips at
the storage layer). There is a small CPU overhead on every I/O operation.

To disable checksums, set `initdb_args` to an empty string **before the first
start** (checksums can only be set at `initdb` time).

## Authentication

The default authentication method is `scram-sha-256` (PostgreSQL 14+). To change authentication methods, use a custom `pg_hba.conf` (see
_Advanced Configuration_ below).

> **Warning**: Do not set `POSTGRES_HOST_AUTH_METHOD=trust` via `extra_args`
> or a custom `pg_hba.conf` unless you have network-level access controls in
> place â€” this allows passwordless access to the database.

## Advanced Configuration

For settings not covered by the `extra_args` list, you can provide custom
`postgresql.conf` and/or `pg_hba.conf` files via the **addon_config** mount.

The addon_config directory on the host is:

```
/addon_configs/local_postgresql18/
```

Place your custom files there:

- `/addon_configs/local_postgresql18/postgresql.conf` â€” replaces the
  generated `postgresql.conf`
- `/addon_configs/local_postgresql18/pg_hba.conf` â€” replaces the generated
  `pg_hba.conf`

If present, these files are passed to postgres via `-c config_file=` and
`-c hba_file=` at startup.

### Common `extra_args` Examples

In the HA UI, each entry has a **name** and **value** field:

| Name | Value | Effect |
|---|---|---|
| `max_connections` | `200` | Raises max concurrent connections |
| `shared_buffers` | `256MB` | Sets shared memory buffer size |
| `work_mem` | `16MB` | Per-operation memory limit |
| `log_statement` | `all` | Logs every statement |
| `listen_addresses` | `192.168.1.10` | Binds to a specific IP |
